核心資料流（每輪對話）

1. User message 進入 Runner 2. Runner 調用 ManagedSessionService.Get() → 返回 [summary_events + active_events]（壓縮後的 view） 3. Runner 組裝 LLMRequest（system prompt + session events + user msg） 4. MemoryPlugin.BeforeModelCallback 攔截 ├─ 從 req 讀取 content 大小（粗估） ├─ 判斷 memory pressure（S3） ├─ 如需壓縮 → 通知 ManagedSessionService 執行壓縮 │ └─ ManagedSessionService 用 Compress Agent 執行（獨立 Runner） │ └─ 更新內部 summaries + active_events └─ 確保 req.Contents 反映最新的壓縮 view 5. LLM API call（Gemini） 6. MemoryPlugin.AfterModelCallback 攔截 ├─ 讀取 resp.UsageMetadata（精確 token 計數） ├─ 更新 MemoryLayout 的 usage 數據 ├─ 首輪校正 PINNED segment（S2） └─ 記錄 metrics 7. Runner 自動 append event 到 Session → ManagedSessionService 攔截 append，更新 active_events 追蹤
